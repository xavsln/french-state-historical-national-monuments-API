French State Historical National Monuments - API

App description:

This App main features include:

  • Return a list of all monuments to the user
  • Return data (name, wikipedia link, type, latitude, longitude, city location, department location, region location, remarks) about a single monument by monumentID to the user
  • Allow new users to register
  • Allow users to update their user info (username, password, email, date of birth)
  • Allow users to add a monument to their list of favorites
  • Allow users to remove a monument from their list of favorites
  • Allow existing users to deregister

Technologies used and dependencies:

  • Node.js
  • Express

API endpoints documentation:

Request description Endpoint URL and query parameters HTTP Method Request body data format Response body data format
Return a list of all monuments to the user /monuments GET None A JSON object holding data about all the monuments
Return data (id, name, wikipedia link, type, latitude, longitude, city location, department location, region location, remarks) about a single monument by monumentID to the user /monuments/[monumentId] GET None A JSON object holding data about a single monument, containing an ID, name, wikipedia link, type, latitude, longitude, city location, department location, region location, remarks. Example: { "id": "1", "name": "Amiens Cathedral, towers", "wikipediaLinkEn": "https://en.wikipedia.org/wiki/Amiens_Cathedral", "type": "Cathedral", "latitude": "46.141736", "longitude": "0.404965", "cityLocation": "Amiens", "departmentLocation": "Somme", "regionLocation": "Picardy", "remarks": null }
Return data (id, name, wikipedia link, type, latitude, longitude, city location, department location, region location, remarks) about all monuments located in a given city to the user /monuments/locations/[cityLocation] GET None A JSON object holding data about all the monument, containing an ID, name, wikipedia link, type, latitude, longitude, city location, department location, region location, remarks about all monuments located in a given city. Example: { "id": "1", "name": "Amiens Cathedral, towers", "wikipediaLinkEn": "https://en.wikipedia.org/wiki/Amiens_Cathedral", "type": "Cathedral", "latitude": "46.141736", "longitude": "0.404965", "cityLocation": "Amiens", "departmentLocation": "Somme", "regionLocation": "Picardy", "remarks": null }
Return a list of all registered users to the admin /users GET None A JSON object holding data about all the users
Return data about a single registered user to the user /users/[username] GET None A JSON object holding data about a single user
Allow new users to register (ie. Add a new User) /users POST A JSON object holding data about the User to add, structured like: { "name" : "John Doe", "favoriteMonuments" : ["Arc de Triomphe", "Château d'Angers"] } A JSON object holding data about the User that was added, including and ID: { "id": "1", "name" : "John Doe", "favoriteMonuments" : ["Arc de Triomphe", "Château d'Angers"] }
Allow users to update their user info (username, password, email, date of birth) /users/[username] PUT A JSON object holding data about the User's name to update, structured like: { "name" : "John Doe" } A JSON object holding data about the User that was updated, including an ID: { "id": "1", "Username" : "Updated John Doe", "Password": "12345", "Email": "john@doe.com", "Birthday": "1985-01-01", "favoriteMonuments" : ["62c82486d2ff5d648ef1538c", "62c823b0d2ff5d648ef1538b", "62c82265d2ff5d648ef15389"] }
Allow users to add a monument to their list of favorites (showing only a text that a monument has been added) /users/[username]/monuments/[monumentName] POST None A text message indicating whether the monument was successfully added to the User's favorite monument list.
Allow users to remove a monument from their list of favorites (showing only a text that a monument has been removed) /users/[username]/monuments/[monumentName] DELETE None A text message indicating whether the monument was successfully removed from the User's favorite monument list.
Allow existing users to deregister (showing only a text that a user email has been removed). (ie. Delete a User from the list of Users) /users/[username] DELETE None A text message indicating whether the User was successfully removed.
